---
description: 将当前会话的关键进展沉淀到 Obsidian 知识库
---

请执行以下操作：

1. **加载 skill**: 加载 `knowledge-save` skill，获取完整的知识库结构规范和工作流程。

2. **回顾会话内容**: 回顾当前会话中的所有关键进展、知识点、结论和方法。

3. **确定沉淀方向**:
   - 如果用户提供了参数: $ARGUMENTS，以此为沉淀的内容方向
   - 如果没有提供参数，自行分析会话中值得沉淀的关键内容

4. **执行沉淀**: 严格按照 `knowledge-save` skill 中定义的工作流程（步骤 1~6）执行知识沉淀。

5. **输出摘要**: 完成后输出操作摘要，列出所有创建和修改的文件。

**重要提醒**:
- 优先更新已有文档，避免子文档泛滥
- 新建 topic 前必须询问用户确认
- tags 控制在 2~4 个
- wikilink 使用带路径前缀的格式
