---
description: 将当前会话的关键进展沉淀到 Obsidian 知识库
---

请通过 Task 工具调用 `knowledge-save` subagent 来执行知识沉淀。

1. **准备内容摘要**（最关键的一步）:

   回顾当前会话的**完整过程**，整理为详尽的摘要。subagent 无法访问你的会话上下文，只能看到你传入的摘要，因此摘要质量直接决定沉淀质量。

   摘要必须包含：
   - **背景**：用户要解决什么问题，起始状态是什么
   - **过程**：尝试了哪些方案，每个方案的思路、遇到的问题、失败原因
   - **关键细节**：涉及的文件路径、函数名、配置项、命令、错误信息
   - **决策点**：为什么选 A 不选 B，各方案的权衡取舍
   - **结论**：最终方案是什么，做了哪些具体修改
   - **技术知识点**：发现的 API 用法、框架机制、系统行为等可复用知识

2. **确定沉淀方向**:
   - 如果用户提供了参数: $ARGUMENTS，以此为沉淀的内容方向
   - 如果没有提供参数，自行分析会话中值得沉淀的关键内容

3. **调用 subagent**: 使用 Task 工具，`subagent_type` 设为 `knowledge-save`，在 `prompt` 中传入内容摘要和方向指示。
   - 如果之前已经调用过 knowledge-save subagent，**必须传入之前的 task_id** 来复用上下文。
   - 如果是首次调用，不传 task_id。

4. **处理返回结果**:
   - 将 subagent 返回的操作摘要展示给用户
   - 如果 subagent 提到"需要新建 topic"，询问用户确认，然后将确认结果通过 task_id 复用再次调用 subagent

**重要**:
- 不要自己执行沉淀操作，不要加载 knowledge-save skill
- 记住 subagent 返回的 task_id，后续调用时复用
